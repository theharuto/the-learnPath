
> “`save()` needs a `User`, so why are we passing a captor instead of a `User`?”

That’s the key misunderstanding.

---

### What’s actually happening here

```java
verify(repository).save(userCaptor.capture());
```

You are **not calling `save()` yourself**.

Instead, you’re telling Mockito:

> “Verify that `save()` was called earlier, and when it was called, **capture the argument that was passed**.”

---

### So what is `ArgumentCaptor` doing?

`ArgumentCaptor` acts like a **hook** or **interceptor**.

* It does NOT replace the `User`
* It does NOT get passed into your real method
* It simply **captures whatever argument was actually used during the real call**

---

### Flow of your test

#### 1. Real method runs

```java
userService.register(10L);
```

Inside this:

```java
repository.save(new User(10L, "ACTIVE"));
```

👉 A real `User` object is created and passed

---

#### 2. Verification phase

```java
verify(repository).save(userCaptor.capture());
```

Now Mockito:

* Looks at the past invocation of `save()`
* Grabs the argument that was passed
* Stores it inside `userCaptor`

---

#### 3. You inspect it

```java
User captured = userCaptor.getValue();
```

Now you can assert:

```java
assertEquals(10L, captured.getId());
```

---

### Why not just use `any()`?

You could do this:

```java id="l3k9zp"
verify(repository).save(any());
```

But that only checks:

> “Was `save()` called?”

It does **not** check:

* What object was passed
* Whether its fields are correct

---

### Why `ArgumentCaptor` is useful

It lets you test:

> “Did my method construct the correct object before passing it?”

That’s especially useful when:

* Object is created inside the method
* You don’t have direct access to it

---

### Simple analogy

* `any()` → “I don’t care what you passed”
* `eq(user)` → “You must pass exactly this object”
* `ArgumentCaptor` → “Show me what you passed, I’ll inspect it”

---

### Key takeaway

You’re not passing a captor **instead of** a `User`.

You’re saying:

> “When `save(User)` was called, capture that `User` so I can examine it.”

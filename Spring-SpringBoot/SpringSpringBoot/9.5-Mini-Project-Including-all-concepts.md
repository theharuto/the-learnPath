
# 4️⃣ Profile-Based Implementations

## Dev Version

```java
package com.example.service;

import org.springframework.context.annotation.Profile;
import org.springframework.stereotype.Service;

@Service
@Profile("dev")
public class DevPaymentService implements PaymentService {

    @Override
    public void processPayment(String orderId) {
        System.out.println("Dev payment processing for: " + orderId);
    }
}
```

---

## Prod Version

```java
package com.example.service;

import org.springframework.context.annotation.Profile;
import org.springframework.stereotype.Service;

@Service
@Profile("prod")
public class ProdPaymentService implements PaymentService {

    @Override
    public void processPayment(String orderId) {
        System.out.println("Real payment gateway called for: " + orderId);
    }
}
```

Only one will load depending on active profile.

# 5️⃣ OrderService (Constructor DI + @Value)

```java
package com.example.service;

import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Service;

@Service
public class OrderService {

    private final PaymentService paymentService;
    private final String appName;

    public OrderService(
            PaymentService paymentService,
            @Value("${app.name}") String appName) {
        this.paymentService = paymentService;
        this.appName = appName;
    }

    @PostConstruct
    public void init() {
        System.out.println("OrderService initialized for app: " + appName);
    }

    public void placeOrder(String orderId) {
        paymentService.processPayment(orderId);
    }

    @PreDestroy
    public void cleanup() {
        System.out.println("Shutting down OrderService...");
    }
}
```

Demonstrates:

* Constructor DI
* Property injection
* Lifecycle hooks
* Loose coupling

# 7️⃣ Main Application (Manual Context)

```java
package com.example;

import com.example.config.AppConfig;
import com.example.service.OrderService;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class MainApp {

    public static void main(String[] args) {

        AnnotationConfigApplicationContext context =
                new AnnotationConfigApplicationContext();

        context.getEnvironment().setActiveProfiles("dev");

        context.register(AppConfig.class);
        context.refresh();

        OrderService orderService = context.getBean(OrderService.class);
        orderService.placeOrder("ORD-123");

        context.close();
    }
}
```

Key concepts shown:

* ApplicationContext
* Profile activation
* Bean retrieval
* Graceful shutdown (triggers @PreDestroy)

# ⚙ What Happens Internally (High-Level)

Startup:

```
1. Create ApplicationContext
2. Activate profile
3. Scan components
4. Register beans
5. Instantiate singletons
6. Inject dependencies
7. Run @PostConstruct
```

Shutdown:

```
1. Call @PreDestroy
2. Release resources
```

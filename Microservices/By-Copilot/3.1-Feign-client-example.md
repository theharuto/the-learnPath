# 🎯 Goal of the Example

We will demonstrate:

*   Two microservices:
    1.  **User Service** (provider)
    2.  **Order Service** (consumer)
*   **Order Service calls User Service using Feign**
*   **Service discovery via Eureka**
*   **Automatic JSON serialization/deserialization**
*   **No RestTemplate / WebClient code**

***

# 1️⃣ High‑Level Architecture (Before Code)

    +-------------------+         +-------------------+
    |  Order Service    | ----->  |  User Service     |
    |                   | Feign   |                   |
    |  (Feign Client)   | HTTP    |  REST Controller  |
    +-------------------+         +-------------------+
              |
              | Service name lookup
              v
          +-------------------+
          |  Eureka Server    |
          |  (Service Registry)
          +-------------------+

***

# 2️⃣ User Service (The Service Being CALLED)

## Step 2.1 — What is User Service?

User Service:

*   Exposes REST APIs
*   Registers itself with **Eureka**
*   Returns user data in **JSON**

***

## Step 2.2 — User Entity (Response Object)

```java
public class User {
    private Long id;
    private String name;
    private String email;

    // getters and setters
}
```

### ✅ Why this class exists

*   JSON from HTTP response → **Java object**
*   Feign will deserialize JSON into this class automatically

***

## Step 2.3 — REST Controller in User Service

```java
@RestController
@RequestMapping("/users")
public class UserController {

    @GetMapping("/{id}")
    public User getUserById(@PathVariable Long id) {
        User user = new User();
        user.setId(id);
        user.setName("Mohammed");
        user.setEmail("mohammed@example.com");
        return user;
    }
}
```

### ✅ What actually happens here

*   This is a **normal Spring MVC controller**
*   Returns a `User` object
*   Spring converts it to JSON automatically

Example HTTP response:

```json
{
  "id": 1,
  "name": "Mohammed",
  "email": "mohammed@example.com"
}
```

***

## Step 2.4 — User Service registers with Eureka

In `application.properties`:

```properties
spring.application.name=user-service
eureka.client.service-url.defaultZone=http://localhost:8761/eureka
```

### ✅ Important concept: service name

*   `user-service` is the **logical name**
*   No IP or port is hard‑coded anywhere

***

# 3️⃣ Order Service (The Service Making the CALL)

## Step 3.1 — What is Order Service?

Order Service:

*   Needs user info to place an order
*   Calls User Service via **Feign**
*   Does NOT know IP or port of User Service

***

# 4️⃣ The Feign Client (CORE OF THE DEMO)

## ✅ This is the MOST IMPORTANT PART

### Step 4.1 — What is a Feign Client?

A **Feign Client** is:

*   A **Java interface**
*   Annotated with `@FeignClient`
*   Represents a **remote HTTP service**

***

## Step 4.2 — Feign Client Interface

```java
@FeignClient(name = "user-service")
public interface UserClient {

    @GetMapping("/users/{id}")
    User getUserById(@PathVariable("id") Long id);
}
```

***

## 🧠 Deep Explanation (Line by Line)

### ✅ `@FeignClient(name = "user-service")`

Means:

> “This interface represents a remote service registered in Eureka as `user-service`”

*   Feign will:
    *   Ask Eureka for available instances
    *   Pick one instance
    *   Call it using HTTP

***

### ✅ `@GetMapping("/users/{id}")`

Means:

> “This Java method maps to an HTTP GET request”

Feign converts method call ↓  
into HTTP call ↓

    GET http://<resolved-ip>:<port>/users/1

***

### ✅ Method return type: `User`

Means:

*   Feign will:
    *   Receive JSON
    *   Convert JSON → `User` object
    *   Return it as method result

✅ **No ObjectMapper**
✅ **No JSON parsing**
✅ **No HTTP code written**

***

# 5️⃣ Enable Feign in Order Service

## Step 5.1 — Main Application Class

```java
@SpringBootApplication
@EnableFeignClients
public class OrderServiceApplication {

    public static void main(String[] args) {
        SpringApplication.run(OrderServiceApplication.class, args);
    }
}
```

### ✅ Why `@EnableFeignClients` exists

It tells Spring:

> “Search for interfaces annotated with `@FeignClient`  
> and generate runtime implementations for them”

***

# 6️⃣ Using the Feign Client (VERY IMPORTANT)

## Step 6.1 — Order Service calls User Service

```java
@Service
public class OrderService {

    private final UserClient userClient;

    public OrderService(UserClient userClient) {
        this.userClient = userClient;
    }

    public void placeOrder(Long userId) {

        // 👇 THIS LOOKS LIKE A LOCAL METHOD CALL
        User user = userClient.getUserById(userId);

        System.out.println("Placing order for user: " + user.getName());
    }
}
```

***

## 💥 What ACTUALLY happens at runtime

When this line runs:

```java
User user = userClient.getUserById(userId);
```

### Behind the scenes (step‑by‑step)

1.  This is **NOT a normal Java object**
2.  `userClient` is a **Feign proxy**
3.  Method call is intercepted
4.  Feign:
    *   Looks up `user-service` in Eureka
    *   Gets list of instances
    *   Picks one (round‑robin)
5.  Builds HTTP request
6.  Sends HTTP call
7.  Receives JSON response
8.  Deserializes JSON → `User`
9.  Returns object

✅ To you, it feels like a local method  
✅ In reality, it is a network call

***

# 7️⃣ Why This Is Powerful (Compared to RestTemplate)

| Without Feign   | With Feign       |
| --------------- | ---------------- |
| Write HTTP code | Write interface  |
| Handle JSON     | Automatic        |
| Manage URLs     | Service name     |
| Error‑prone     | Clean & readable |
| Boilerplate     | Declarative      |

***

# 8️⃣ Final Mental Model (Very Important)

> **Feign turns HTTP calls into Java method calls, while Spring handles service discovery, load balancing, serialization, and deserialization automatically.**

# 📦 Goal

```text
Swagger UI → Click "Authorize" → Enter JWT → All requests include Authorization header
```

---

# 🔹 1. Mental Model

```text
Client request →
Authorization: Bearer <JWT>
```

Swagger needs to:

1. Know this header exists
2. Provide UI to enter token
3. Attach it to requests

# 🔹 2. Add Security Scheme (Global Config)

---

## Code

```java id="jwt1"
import io.swagger.v3.oas.annotations.OpenAPIDefinition;
import io.swagger.v3.oas.annotations.enums.SecuritySchemeType;
import io.swagger.v3.oas.annotations.info.Info;
import io.swagger.v3.oas.annotations.security.SecurityScheme;
import org.springframework.context.annotation.Configuration;

@Configuration
@OpenAPIDefinition(
    info = @Info(
        title = "Secure API",
        version = "1.0",
        description = "API with JWT authentication"
    )
)
@SecurityScheme(
    name = "bearerAuth",
    type = SecuritySchemeType.HTTP,
    scheme = "bearer",
    bearerFormat = "JWT"
)
public class OpenApiConfig {
}
```

---

## 🔹 What this does

* Defines authentication type
* Tells Swagger:

```text
"This API uses Authorization: Bearer <token>"
```

# 🔹 3. Apply Security to Endpoints

---

## Option 1: Apply on Controller

```java id="jwt2"
import io.swagger.v3.oas.annotations.security.SecurityRequirement;

@RestController
@RequestMapping("/resources")
@SecurityRequirement(name = "bearerAuth")
public class ResourceController {

    @GetMapping
    public String get() {
        return "secured data";
    }
}
```

---

## Option 2: Apply per method

```java id="jwt3"
@Operation(summary = "Secure endpoint")
@SecurityRequirement(name = "bearerAuth")
@GetMapping("/secure")
public String secure() {
    return "secured";
}
```

# 🔹 5. Important Details (Production)

---

## 1. Do NOT include "Bearer " manually in config

Swagger handles it:

```text
You enter: eyJhbGciOiJIUzI1NiIs...
Swagger sends: Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

---

## 2. Public vs Secured APIs

You can mix:

```java
@GetMapping("/public")
public String publicApi() { return "no auth"; }
```

```java
@SecurityRequirement(name = "bearerAuth")
@GetMapping("/private")
public String privateApi() { return "auth required"; }
```

---

## 3. Spring Security Integration

Swagger does NOT enforce security.
It only sends headers.

👉 Actual validation is done by:

* Spring Security filters
* JWT verification logic

# 🔹 6. Common Mistakes

---

❌ Forgetting `@SecurityRequirement`
→ Swagger won’t attach token

---

❌ Wrong scheme name mismatch

```java
@SecurityScheme(name = "bearerAuth")
@SecurityRequirement(name = "jwtAuth") // ❌ mismatch
```

---

❌ Expecting Swagger to validate JWT
→ It only sends header, not validate

# рЯФє 5. Validation Groups (THE REAL DIFFERENCE)

---

## Problem

Different rules for:

* Create
* Update

---

## Example

```java
public interface CreateGroup {}
public interface UpdateGroup {}
```

---

## DTO

```java
public class UserDTO {

    @NotNull(groups = UpdateGroup.class)
    private Long id;

    @NotBlank(groups = CreateGroup.class)
    private String name;
}
```

---

## Controller

```java
@PostMapping
public void create(
    @Validated(CreateGroup.class) @RequestBody UserDTO dto) {}

@PutMapping
public void update(
    @Validated(UpdateGroup.class) @RequestBody UserDTO dto) {}
```

---

## рЯФ• Key Point

рЯСЙ This **ONLY works with `@Validated`**

# рЯФє 11. Clean Summary (Interview Ready)

> `@Valid` is a standard Jakarta validation annotation used for basic validation, while `@Validated` is a Spring-specific annotation that supports advanced features like validation groups and method-level validation through AOP. In production, `@Valid` is used for simple cases, and `@Validated` is preferred when group-based or advanced validation is required.

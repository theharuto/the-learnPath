# 🔹 1. All Relevant Annotations (Request & Response Handling)

## ✅ Core Request Handling

### `@RequestBody`

* Binds HTTP body → Java object
* Uses `HttpMessageConverter` (Jackson)

---

### `@ResponseBody`

* Converts return value → HTTP response body
* Already included in `@RestController`

---

### `@RequestParam`

* Extract query parameters

```text
GET /items?page=1
```

---

### `@PathVariable`

* Extract path values

```text
GET /items/{id}
```

---

### `@RequestHeader`

* Read request headers

```java
@GetMapping
public String get(@RequestHeader("Authorization") String token) {
    return token;
}
```

---

### `@RequestPart`

* Used in multipart requests (file + JSON)

---

---

## ✅ Response Handling

### `@ResponseStatus`

* Sets HTTP status without `ResponseEntity`

```java
@ResponseStatus(HttpStatus.CREATED)
@PostMapping
public Resource create() {
    return new Resource();
}
```

⚠️ Limited control → prefer `ResponseEntity` in production

---

### `ResponseEntity` (not annotation but critical)

* Full control over:

  * Status
  * Headers
  * Body

---

---

## ✅ Serialization / Conversion (Advanced but important)

### `@JsonProperty`

* Rename fields in JSON

### `@JsonIgnore`

* Hide fields from response

### `@JsonInclude`

* Control null handling

👉 These are from **Jackson**, heavily used in production

# 🔹 2. HTTP Status Codes + Spring Usage

---

## ✅ Common Status Codes (You MUST know these)

| Code                      | Meaning          | When Used                  |
| ------------------------- | ---------------- | -------------------------- |
| 200 OK                    | Success          | GET, successful operations |
| 201 CREATED               | Resource created | POST                       |
| 204 NO CONTENT            | Success, no body | DELETE                     |
| 400 BAD REQUEST           | Invalid input    | Validation failure         |
| 401 UNAUTHORIZED          | Auth required    | Missing/invalid auth       |
| 403 FORBIDDEN             | Access denied    | No permission              |
| 404 NOT FOUND             | Resource missing | Invalid ID                 |
| 409 CONFLICT              | State conflict   | Duplicate / version issue  |
| 500 INTERNAL SERVER ERROR | Server failure   | Unexpected errors          |

---

## ✅ Spring Enum

Use:

```java
org.springframework.http.HttpStatus
```

---

## Examples

```java
HttpStatus.OK
HttpStatus.CREATED
HttpStatus.BAD_REQUEST
HttpStatus.NOT_FOUND
```

---

## With ResponseEntity

```java
return ResponseEntity.status(HttpStatus.CREATED).body(data);
```

---

## Shortcut Methods

```java
ResponseEntity.ok(data)
ResponseEntity.notFound().build()
ResponseEntity.noContent().build()
```

# 🔹 3. Custom Headers (Deep Understanding)

---

## Short Explanation

> Headers = metadata about request/response (not the main data)

---

## Mental Model

```text
HTTP Response =
{
  status,
  headers,
  body
}
```

---

## Example (Multiple Headers)

```java
@GetMapping("/{id}")
public ResponseEntity<Resource> get() {

    Resource res = new Resource(1, "data");

    return ResponseEntity.ok()
        .header("X-Request-Id", "abc-123")
        .header("X-Source", "internal-service")
        .header("Cache-Control", "no-cache")
        .body(res);
}
```

---

## Alternative (Using HttpHeaders)

```java
@GetMapping("/{id}")
public ResponseEntity<Resource> get() {

    HttpHeaders headers = new HttpHeaders();
    headers.add("X-Request-Id", "abc-123");
    headers.add("X-Source", "internal-service");

    return new ResponseEntity<>(new Resource(1, "data"), headers, HttpStatus.OK);
}
```

# 🔹 4. Why Custom Headers Matter (Production Use Cases)

---

## 1. **Tracing & Debugging**

```text
X-Request-Id: abc-123
```

* Track request across microservices
* Essential for distributed systems

---

## 2. **Pagination Metadata**

```text
X-Total-Count: 100
X-Page: 2
```

* Client knows:

  * total records
  * current page

---

## 3. **Rate Limiting**

```text
X-Rate-Limit-Remaining: 50
```

* Prevent abuse
* Inform client limits

---

## 4. **Caching Control**

```text
Cache-Control: no-cache
```

* Improve performance
* Avoid stale data

---

## 5. **Security**

```text
Authorization: Bearer token
```

* Authentication & authorization

---

## 6. **Resource Location (VERY IMPORTANT)**

After creation:

```text
Location: /resources/10
```

✔ Standard REST practice

# 🔹 5. Production-Level Rules (Must Remember)

---

## Rule 1: Always use correct status codes

❌ Everything → 200
✔ Use semantics properly

---

## Rule 2: Use `ResponseEntity` for flexibility

* Needed for:

  * headers
  * custom status

---

## Rule 3: Headers are not optional in real systems

* Critical for:

  * observability
  * performance
  * client communication

---

## Rule 4: Never expose internal fields blindly

Use:

* DTOs (next topic)
* Jackson annotations
